SI_1. Úkol – Deklarace záměru (3 body) Deklarace záměru projektu – Informační systém cestovní kanceláře

Cíl a vize projektu Cílem projektu je vytvořit webovou aplikaci pro správu autobusových zájezdů pořádaných cestovní kanceláří. Systém sjednotí evidenci zájezdů, přihlášky účastníků, platby a komunikaci s klienty, aby se snížila chybovost a administrativní zátěž. Menší cestovní kanceláře často vedou agendu zájezdů a klientů pomocí tabulek v aplikaci Excel, což je nepřehledné, chybové a přináší riziko ztráty dat. Projekt nabídne centralizované webové řešení, které zajistí přehlednost, bezpečnost a snadnou správu dat.
Kontext a události Cestovní kancelář pořádá autobusové zájezdy (1 zájezd = 1 autobus). Systém musí pokrýt tyto události: • Přihlášení na zájezd – účastník rezervuje místo do termínu T_reg. • Zaplacení zájezdu – do termínu T_pay; po uplynutí systém rezervaci zruší. • Odhlášení ze zájezdu – účastník se může odhlásit do stanoveného termínu. • Zrušení zájezdu – pracovník CK zájezd zruší a systém informuje účastníky e-mailem.
Uživatelské role • Občan – neregistrovaný uživatel, který prohlíží nabídku zájezdů (destinace, termín, cena, volná místa). • Účastník – registrovaný uživatel, který se přihlašuje nebo odhlašuje na zájezdy, provádí platby a sleduje historii přihlášek. • Pracovník CK – vytváří, upravuje a ruší zájezdy, kontroluje platby, rozesílá hromadné zprávy klientům. • Administrátor – spravuje systém, uživatele a generuje přehledy o zájezdech.
Hlavní funkce systému • Vyhledávání a filtrování zájezdů podle destinace, data nebo ceny. • Registrace a přihlášení uživatele. • Tvorba, úprava a rušení přihlášek. • Evidence plateb a kontrola termínů T_pay. • Automatické notifikace e-mailem. • Správa uživatelů a přehledy o zájezdech.
Použité technologie Frontend: HTML5, CSS3, Bootstrap 5 Backend: PHP 8, MySQL Vývojové prostředí: Visual Studio Code Správa verzí: GitHub
